# 棋盘玩法规则文档

> 纯规则说明，不含代码、文件路径、技术实现。

---

## 1. 棋盘基础规则

### 棋盘是什么

棋盘是一个二维网格平面，弹球在上面移动，玩家在上面放置各种对象来影响弹球的路径。

### 棋盘有多大

棋盘由 **7 行 × 7 列 = 49 个格子** 组成。每个格子是正方形，等距排列。

### 棋盘的两个区域

棋盘分为两个功能区域：

| 区域 | 范围 | 格子数 | 用途 |
| -- | -- | -- | -- |
| 内棋盘 | 第 1~5 行 × 第 1~5 列 | 25 格 | 弹球移动、玩家放置对象 |
| 管道 | 外圈一周（第 0 行、第 6 行、第 0 列、第 6 列） | 24 格 | 弹球高速循环通道 |

其中有一个特殊格子——**入口格**。入口坐标是可配置参数，当前暂定在第 3 行第 0 列（左边缘中间），是弹球进入内棋盘的位置。

管道和入口格都属于外圈，**玩家不能在上面放置任何对象**。

### 格子坐标

- 左上角是坐标 (行 0, 列 0)，向下是行增加，向右是列增加
- 内棋盘的可操作范围：行 1~5，列 1~5
- 管道范围：行 0 或 6，列 0~6；列 0 或 6，行 0~6

### 格子的三种占用状态

- **空**：格子上没有任何对象，弹球可以自由通过
- **占用但可通过**：格子上有对象，但弹球依然可以进入（如减速器、混沌门、黑洞、残骸）
- **占用且阻挡**：格子上有对象，弹球无法进入，碰到后会反弹（如转向器的不连通方向、旋转器的不连通方向、冰块、武器本体）

### 棋盘在游戏中的作用

棋盘是游戏的**唯一战场**。所有互动——弹球移动、转向、充能、武器开火——都发生在棋盘格子上。玩家策略的核心就是在内棋盘 25 格中合理安排对象布局，引导弹球按期望的路径移动。

## 2. 棋盘 Entity 规则

Entity 指玩家可以放置在棋盘内棋格上的对象。当前共有 9 种有效类型。

### Entity 总览表

| 对象类型 | 是否占用格子 | 可否被弹球经过 | 弹球经过时的效果 | 触发后自身是否变化 | 可否被移除 |
| -- | -- | -- | -- | -- | -- |
| 空格 | 不占用 | 是 | 无效果，正常通过 | — | — |
| 转向器 | 占用 | 部分方向可通过 | 改变弹球方向 + 加速 | 状态不变 | 可 |
| 旋转器 | 占用 | 部分方向可通过 | 改变弹球方向 + 加速 | 离开后自转一个方位 | 可 |
| 减速器 | 占用 | 是 | 弹球移动速度大幅降低 | 状态不变 | 可 |
| 冰块 | 占用 | 否 | 弹球被弹回，冰块耐久减 1 | 耐久归零后消失 | 可 |
| 混沌门 | 占用 | 是 | 弹球方向随机变为上下左右之一 | 状态不变 | 可 |
| 黑洞 | 占用 | 是 | 弹球被传送回入口，方向和速度重置 | 状态不变 | 可 |
| 残骸 | 占用 | 是 | 无效果 | 可被武器清除 | 可 |
| 石头 | 占用 | 否 | 弹球被弹回 | 状态不变，永不损毁 | 不可 |
| 武器 | 占用 | 否（弹球不能进入武器格） | 弹球碰到武器格时反弹 | 通过"尾巴"充能，满能自动开火 | 可 |

### 空格说明

空格不是一种 Entity，是棋盘格子的默认状态。弹球在空格上移动时，方向不变、速度不变、无任何特殊效果。

### 各 Entity 详细对比

| 对比维度 | 转向器 | 旋转器 | 减速器 | 冰块 | 混沌门 | 黑洞 | 残骸 | 石头 | 武器 |
| -- | -- | -- | -- | -- | -- | -- | -- | -- | -- |
| 弹球能否进入 | 仅连通方向可进 | 仅连通方向可进 | 可 | 不可 | 可 | 可 | 可 | 不可 | 不可 |
| 进入时改变方向 | 是 | 是 | 否 | — | 是（随机） | 否 | 否 | — | — |
| 进入时改变速度 | 是（等级越高越快） | 是（等级越高越快） | 是（大幅减速） | — | 否 | 否 | 否 | — | — |
| 离开时触发效果 | 否 | 是（自转） | 否 | — | 否 | 否 | 否 | — | — |
| 弹球碰壁时效果 | 反弹 | 反弹 | — | 反弹+扣耐久 | — | — | — | 反弹 | 反弹 |

## 3. 弹球移动规则

### 弹球从哪里进入棋盘

弹球始终从**入口格**进入棋盘。入口坐标是可配置参数，当前暂定为棋盘左边缘中间位置（行 3, 列 0）。

### 弹球初始方向

弹球从入口进入时，方向为**向右**，直接进入内棋盘。

### 弹球如何在格子之间移动

弹球一次走一格，沿当前方向走到相邻格子。移动节奏由"步长"控制——正常情况下一步约 0.8 秒（可以调整为 0.4 秒）。

每一步分为 5 个连续阶段：
1. **当前格生效**：应用所在格子上对象的进入效果（如转向、减速、黑洞传送）
2. **移动**：决定下一格并移动过去
3. **武器触发**：检查是否经过了武器的触发区域，充能并开火
4. **离开效果**：应用离开上一格时触发的效果
5. **记录历史**：记录这一步的移动轨迹

### 弹球在内棋盘的移动

弹球在内棋盘中，每次按当前方向走一格：

- 如果下一格是空的：直接进入
- 如果下一格有可通过的对象：进入该格，触发该对象的进入效果
- 如果下一格有不可通过的对象：弹球**反弹**（方向反转），停在原地。如果该对象是冰块，冰块耐久减 1
- 如果下一步会走出内棋盘边界：判断边界外格子是否属于管道

### 弹球在管道中的移动

当弹球在内棋盘边界走出去，且边界外格子恰好在管道上时，弹球进入管道：

- 弹球沿管道路径**顺时针**高速移动（速度是内棋盘的 10 倍）
- 管道共 24 格，围绕内棋盘外围一周
- 弹球在管道内一格一格前进，方向由管道自动决定
- 当弹球回到入口格位置时，**自动切回内棋盘**，方向重置为向右，速度恢复为正常
- 如果弹球出界但边界外不是管道格子，弹球反弹

### 弹球如何判断进入哪个格子

进入一个格子取决于"是否可以通过"：
- 空格总是可通过
- 有对象的格子，取决于该对象是否允许弹球从当前方向进入
  - 转向器/旋转器：只有从特定方向（连通方向）进入才允许
  - 武器本体/冰块/石头：从不允许进入
  - 其他对象：总是允许进入

### 弹球什么时候转向

弹球的转向发生在**到达当前格中心点、准备走下一步时**（即进入新一格的那个瞬间）。具体时机如下：

| 触发场景 | 转向时机 | 转向结果 |
| -- | -- | -- |
| 连通转向器 | 弹球到达转向器格中心点、走完当前步的瞬间 | 方向按转向器规则改变，下一步朝新方向移动 |
| 连通旋转器 | 弹球到达旋转器格中心点、走完当前步的瞬间 | 方向按旋转器规则改变，下一步朝新方向移动 |
| 混沌门 | 弹球到达混沌门格中心点、走完当前步的瞬间 | 方向随机变为上下左右之一，下一步朝新方向移动 |
| 碰不可通过对象 | 弹球试图走向被阻挡的格子时（一步的中点附近） | 方向反转（反弹），弹球回到原格 |
| 管道转向 | 弹球到达每个管道格中心点时 | 方向自动按顺时针调整，沿管道路径逐格前进 |
| 管道出口（回到入口格） | 弹球到达入口格中心点时 | 方向重置为向右，切回内棋盘 |

关键规则：弹球在执行完整一步后才会改变方向。一步的"中间点"是弹球从上一格中心点到当前格中心点的移动过程，方向在整步过程中保持不变。

### 弹球什么时候被传送

当弹球进入**黑洞**所在格、到达中心点时：
- 弹球立即被传送到入口格（当前配置为行 3, 列 0）
- 方向重置为向右
- 速度重置为正常速度
- 这一步之后不再继续移动（当前步直接终止）

### 弹球什么时候减速

| 触发场景 | 减速时机 | 减速效果 |
| -- | -- | -- |
| 进入减速器格 | 弹球**一踏入**减速器所在格（从上一步落地到该格的瞬间开始） | 当前整步的耗时为正常步长的 2.5 倍（约 2 秒），弹球在该步内移动缓慢 |

减速只在弹球实际占据减速器格的这一步内生效。离开减速器格后，下一步恢复正常速度。

### 弹球什么时候加速

| 触发场景 | 加速时机 | 速度变化 | 何时恢复 |
| -- | -- | -- | -- |
| 进入转向器/旋转器（连通） | 弹球到达转向器/旋转器格**中心点**、完成当前步的瞬间 | 速度倍率变为对应等级的值（1~5 级对应 1.5x/2.5x/3.5x/4.5x/6x） | 离开转向器/旋转器格后，下一步的速度由新格决定（空格则恢复 1 倍） |
| 进入管道 | 弹球踏入**第一个管道格子**中心点、完成该步的瞬间 | 速度倍率变为 10 倍（远快于内棋盘） | 弹球回到入口格、切回内棋盘时，速度恢复为 1 倍 |
| 回到入口格 | 弹球从管道回到入口格**中心点**的瞬间 | 速度倍率**恢复为初始速度 1 倍**，方向重置为向右 | 此后按内棋盘正常规则运行 |

**关键规则**：速度倍率在弹球进入新一格中心点时生效/切换，不是提前生效也不是滞后生效。弹球在整步移动过程中速度不变，变化发生在步与步之间的"过渡点"（到达格子中心点的瞬间）。

## 4. 转向器规则

### 转向器是什么

转向器是玩家放在内棋盘格子上的 L 形方向偏转装置。弹球从特定方向进入转向器后，方向被改变为另一方向。

### 转向器的四种类型

转向器有 4 种朝向，每种允许的进入方向和对应的出口方向不同：

| 转向器类型 | 符号表示 | 允许的进入方向 | 弹球出口方向 | 是否占格 | 触发后自身状态 |
| -- | -- | -- | -- | -- | -- |
| 右上角型 | └ | 从下方进入 或 从左方进入 | 下方→向右 / 左方→向上 | 是 | 不变 |
| 右下角型 | ┌ | 从上方进入 或 从左方进入 | 上方→向右 / 左方→向下 | 是 | 不变 |
| 左上角型 | ┘ | 从下方进入 或 从右方进入 | 下方→向左 / 右方→向上 | 是 | 不变 |
| 左下角型 | ┐ | 从上方进入 或 从右方进入 | 上方→向左 / 右方→向下 | 是 | 不变 |

### 连通判断详解

转向器的核心规则是"连通"——不是所有方向都能进入转向器。

- 弹球从右方来（向左移动）→ 只能进入"左上"或"左下"型转向器
- 弹球从左方来（向右移动）→ 只能进入"右上"或"右下"型转向器
- 弹球从下方来（向上移动）→ 只能进入"右上"或"左上"型转向器
- 弹球从上方来（向下移动）→ 只能进入"右下"或"左下"型转向器

如果弹球方向与转向器不连通，弹球会被**反弹**（方向反转），不会进入转向器格子。

### 转向器等级与速度

每个转向器有 1~5 级。等级越高，弹球经过时移动越快。所有 Entity 等级共用的颜色规则如下：

| 等级 | 颜色 | 速度倍率 |
| -- | -- | -- |
| 1 | 白 | 1.5 倍 |
| 2 | 绿 | 2.5 倍 |
| 3 | 蓝 | 3.5 倍 |
| 4 | 橙 | 4.5 倍 |
| 5 | 红 | 6.0 倍 |

颜色从低到高：白 → 绿 → 蓝 → 橙 → 红。该颜色规则适用于所有有等级的 Entity（转向器、旋转器、武器等）。

### 旋转器与转向器的区别

旋转器的外观和行为与转向器类似（相同的 4 种朝向、相同的连通规则），但有**一个关键区别**：

- 弹球离开旋转器后，旋转器会**自转一个方位**
- 自转顺序：右上 → 右下 → 左下 → 左上 → 右上（顺时针循环）
- 自转意味着下次弹球经过时，允许进入的方向和出口方向都会变化

转向器则不会自转，放置后永远保持固定朝向。

### 转向器/旋转器的放置规则

- 只能放在内棋盘格子（行 1~5, 列 1~5）上
- 占用格子，放置后该格不能再放其他对象
- 可以随时被玩家移除或替换
- 可以升级（1~5 级）
- 可以在 4 种朝向上循环切换

## 5. 武器规则

### 武器是什么

武器是玩家放在内棋盘格子上的攻击装置。弹球不会直接进入武器本体所在的格子，而是通过经过武器的"尾巴"来给武器充能，充能满后武器自动开火。

### 武器本体

- 武器本体是一个**正方形**，有自己的朝向（上/下/左/右）
- 武器本体占用一个格子
- 弹球**不能进入**武器本体所在的格子。碰到武器格时弹球会反弹
- 武器本体可以在放置时选择朝向，也可以随时切换朝向
- **本版本不做特效**：武器充能和开火仅在控制台以日志形式输出（如"手枪 充能 2/4""手枪 发射"），不伴随屏幕上的视觉特效

### 武器尾巴

- 武器的尾巴位于武器本体朝向的**反方向一格**。尾巴最多可以有 4 个方向（上、下、左、右），即一个武器最多 4 条尾巴
- 例如：武器朝上，其尾巴在武器下方（同一列，行号 +1）
- 例如：武器朝右，其尾巴在武器左方（同一行，列号 -1）
- 尾巴不是一个独立的格子对象——它和武器本体共享同一个武器 Entity
- 弹球经过任意一个尾巴所在格时，都会触发该武器充能
- **尾巴可以伸到管道格子上**：如果武器放在内棋盘边界，其尾巴所在格可能位于管道路径上，弹球在管道中经过该格时同样触发充能

### 武器合成与尾巴继承

武器支持合成升级，合成规则取决于两个武器尾巴的方向是否相同：

**合成前提**：两个**同类型、同等级**的武器。

**合成规则**：

| 合成条件 | 合成结果 |
| -- | -- |
| 两个尾巴**方向相同**的同类型同级武器 | 产生一个等级 +1 的同类型武器，**保留 1 条尾巴**（方向不变） |
| 两个尾巴**方向不同**的同类型同级武器 | 产生一个等级 +1 的同类型武器，**保留 2 条尾巴**（继承两个方向） |

**尾巴数量上限**：一个武器最多拥有 4 条尾巴（对应上、下、左、右四个方向）。尾巴数量达到 4 条后，再合成不再增加尾巴。

**合成示例**：
- 1 级手枪（尾巴朝下） + 1 级手枪（尾巴朝下） → 2 级手枪（尾巴朝下，1 条）
- 2 级手枪（尾巴朝下，1 条） + 2 级手枪（尾巴朝左，1 条） → 3 级手枪（尾巴朝下+朝左，2 条）
- 若继续与另一 3 级手枪（尾巴朝右，1 条）合成 → 4 级手枪（尾巴朝下+朝左+朝右，3 条）
- 若继续与另一 4 级手枪（尾巴朝上，1 条）合成 → 5 级手枪（上下左右，4 条满）

**合成注意事项**：
- 合成后，被合成的两个子武器从棋盘上消失
- 合成产出的武器等级 +1，但充能计数器归零
- 合成后无法降级或拆分

### 弹球与武器本体的交互

弹球碰到武器本体所在格的规则：
- 弹球**不能进入**武器格
- 碰到后弹球**反弹**（方向反转）
- 弹球不会受伤，武器也不会因此受损
- 这次碰撞**不会**给武器充能

### 弹球与武器尾巴的交互

弹球进入武器尾巴所在格的规则：
- 尾巴格可能已经被武器本体"逻辑占用"，但尾巴格依然可以容纳其他 Entity
- 尾巴可以位于内棋盘格子或管道格子上
- 弹球可以正常经过尾巴格（无论在内棋盘还是管道中）
- 弹球每次经过尾巴格，**该武器充能 +1**
- 同一武器在同一步内可以被经过多次（如果弹球在尾巴格上来回移动）

### 武器充能规则

| 武器类型 | 满充能所需次数 | 攻击方式 | 攻击范围 |
| -- | -- | -- | -- |
| 手枪 | 2 次 | 沿朝向直线射击 | 直线，打到第一个目标停止 |
| 炸弹 | 3 次 | 范围爆破 | 以武器为中心的 3×3 区域 |
| 激光 | 4 次 | 沿朝向整条轴线 | 整行（左右朝向）或整列（上下朝向） |
| 闪电 | 5 次 | 全场打击 | 场上所有冰块和残骸 |

### 武器满能量后的行为

1. 充能达到上限后，武器**自动开火**，不需要玩家手动触发
2. 开火后充能**立即归零**
3. 如果同一时刻有多个武器满充能，**依次开火**（先满的先开）
4. 武器开火**不直接影响弹球**的移动（弹球继续正常移动）
5. 武器开火后，可以重新积累充能，进行下一次攻击

### 武器占格规则

| 武器类型 | 本体占格 | 尾巴位置 | 弹球碰本体 | 弹球过尾巴 |
| -- | -- | -- | -- | -- |
| 手枪 | 占用 1 格 | 本体各个朝向的反方向一格，最多 4 条（可伸到管道格） | 反弹 | 充能 +1 |
| 炸弹 | 占用 1 格 | 同上 | 反弹 | 充能 +1 |
| 激光 | 占用 1 格 | 同上 | 反弹 | 充能 +1 |
| 闪电 | 占用 1 格 | 同上 | 反弹 | 充能 +1 |

所有武器的占格和尾巴规则完全一致，区别仅在于充能上限和攻击方式。尾巴数量取决于武器等级和合成历史（1 级 1 条，最多 4 条）。

---

## 6. 武器尾巴充能规则

### 什么是武器尾巴

武器尾巴是武器充能的**触发区域**。它不是棋盘上一个独立的格子对象，而是与武器本体绑定的逻辑位置——位于武器本体朝向相反方向的那一格。

### 尾巴与武器的关系

- 1 级武器只有 1 条尾巴；高等级武器通过合成获得多条尾巴，最多 4 条（上/下/左/右各一）
- 每条尾巴的位置由武器本体的位置和朝向决定（位于对应方向的反方向一格）
- 改变武器本体位置 → 所有尾巴位置跟着变
- 尾巴可以伸到管道格子上，弹球在管道内经过时同样触发充能
- 尾巴不能独立于武器存在

### 尾巴的触发条件

- 弹球**进入**尾巴所在的格子时，触发充能
- 弹球必须**实际走到**尾巴格上才触发（仅仅"朝尾巴方向移动"不算）
- 弹球从任何方向进入尾巴格都触发充能
- 同一弹球在同一步中经过同一个尾巴多次，每次经过都 +1 充能

### 充能累加规则

- 每次经过任意一个尾巴充能 +1
- 高等级武器拥有多个尾巴，弹球经过任意一个尾巴都独立触发充能（每经过一个尾巴 +1）
- 不同弹球的经过都计入同一个充能计数器
- 没有"冷却时间"——连续经过连续充能
- 如果充能已经满了，弹球继续经过尾巴不会溢出（到上限即停，等待开火）

### 充能显示

当前版本武器的充能状态和开火仅通过**控制台日志**输出，不伴随屏幕视觉特效：
- 充能时输出："手枪 充能 1/4"
- 满能开火时输出："手枪 发射"
- 后续正式版本会在武器本体上直观显示充能进度和开火特效

### 充满后开火

充能达到该武器类型的上限后：
1. 充能计数器归零
2. 武器立即执行一次攻击
3. 弹球继续正常移动，不受武器开火影响
4. 武器进入下一轮充能周期

## 7. 棋盘状态变化规则

### 放置对象时

- 目标格子从"空"变为"占用"
- 如果目标格已有对象，旧对象被替换（旧对象消失）
- 新对象以默认状态出现（等级 1、耐久满、充能 0）

### 移除对象时

- 目标格子从"占用"变为"空"
- 对象及其所有状态（等级、充能、耐久）全部清除

### 弹球经过空格时

- 空格状态不变
- 弹球方向不变（无外部影响时）
- 不产生任何棋盘状态变化

### 弹球经过有对象的格子时

- **转向器**：弹球方向改变、速度改变，转向器自身状态不变
- **旋转器**：弹球方向改变、速度改变，旋转器在弹球离开后自转一个方位
- **减速器**：弹球步长变长，减速器自身状态不变
- **混沌门**：弹球方向随机改变，混沌门自身状态不变
- **黑洞**：弹球传送回入口，黑洞自身状态不变
- **残骸**：无效果，残骸自身状态不变
- **武器尾巴**：武器充能 +1

### 弹球碰到不可通过的对象时

- **冰块**：弹球反弹，冰块耐久减 1；耐久归零后冰块消失，该格变为空格
- **石头**：弹球反弹，石头不损耗耐久，永不消失
- **武器本体**：弹球反弹，武器状态不变
- **转向器/旋转器不连通方向**：弹球反弹，对象状态不变

### 武器开火时

- 武器开火可能清除棋盘上的冰块和残骸（**石头不会被任何武器破坏**）：
  - 手枪：清除直线上的第一个残骸或冰块
  - 炸弹：清除 3×3 范围内的残骸和冰块（冰块扣 3 耐久）
  - 激光：清除整行或整列上的残骸和冰块（冰块扣 2 耐久）
  - 闪电：清除场上所有残骸和冰块（冰块扣 2 耐久）
- 被清除的对象消失，所在格子变回空格
- 武器本身的充能归零

### 哪些状态是临时/会变化的

| 状态 | 性质 |
| -- | -- |
| 武器充能 | 临时，弹球经过尾巴累加，满能开火后归零 |
| 冰块耐久 | 受击扣除，归零后冰块永久消失 |
| 旋转器朝向 | 每次弹球离开后改变（自转） |
| 弹球位置/方向/速度 | 持续变化 |
| 棋盘格子的占用状态 | 受放置/移除/冰块消除影响 |
| 转向器朝向/等级 | 仅受玩家操作影响，不自动变化 |
| 减速器/混沌门/黑洞 | 放置后状态固定不变 |
| 石头 | 放置后永久存在，无法移除，无法被武器破坏 |

## 8. 玩家操作规则

> **调试阶段说明**：以下操作规则描述的是当前调试/验证阶段的行为。当前通过调试面板上的按钮完成所有操作，操作不消耗任何资源（沙盒模式）。后续正式游戏中，玩家操作方式（如拖拽、快捷菜单）和资源消耗规则会在此基础上扩展。

### 玩家可以对棋盘做什么

| 操作 | 是否支持 | 规则说明 |
| -- | -- | -- |
| 点击空格子选中 | 支持 | 点击后该格高亮，状态显示区更新 |

| 升级/降级选中对象 | 支持 | 仅转向器和旋转器支持升级（1~5 级循环） |
| 切换转向器朝向 | 支持 | 在 4 种朝向上循环切换 |
| 切换旋转器朝向 | 支持 | 在 4 种朝向上循环切换 |
| 切换武器朝向 | 支持 | 在上下左右 4 个方向循环切换 |

| 清空所有弹球 | 支持 | 所有弹球被移除 |
| 重置棋盘 | 支持 | 恢复到默认初始状态 |
| 合成武器 | 规划中 | 两个同类型同等级武器合成为高一级武器，尾巴继承 |

### 操作成功/失败规则

- **放置成功条件**：选中了内棋盘格子，且选择了要放置的对象类型
- **放置失败条件**：未选中格子，或选中了管道路径上的格子（管道格不可放置对象）
- **升级成功条件**：选中格上有转向器或旋转器（其他对象不支持升级）
- **移除成功条件**：选中格上有对象
- **等级上限**：5 级，再升级回到 1 级
- **朝向循环**：4 种朝向依次切换，无上限

### 当前操作的局限性

- 每格最多放置一个对象（不能叠加）
- 不能同时选中多个格子批量操作
- 不能拖拽对象从一个格子移动到另一个格子（需要先删再放）
- 没有撤销操作
- 放置对象不消耗任何资源（当前是调试沙盒模式）

## 9. 规则摘要

以下是一段可以直接给后续 AI 使用的棋盘玩法规则摘要：

---

## 棋盘玩法规则摘要

### 棋盘如何运行

棋盘是一个 7×7 网格（49 格），分为两个区域：中间的 5×5 内棋盘（25 格，可放置对象）和外圈 24 格管道（弹球高速通道，不可放置对象）。弹球从左边中间的入口格进入内棋盘，在格子间移动，与玩家放置的对象互动。所有游戏行为都发生在这个棋盘上。

### 弹球如何移动

弹球从入口格（可配置，当前暂定左边中间）进入，初始方向向右。每步走一格，沿当前方向前进。空格自由通过。有对象的格子根据对象类型决定弹球能否进入、方向是否改变、速度是否变化。碰到不可通过的对象时弹球反弹。走出内棋盘边界时，如果进入管道则高速顺时针绕一圈后回到入口再次进入，否则反弹。

### 对象如何放置

玩家点击内棋盘格子选中位置，通过按钮选择对象类型放置。每格最多一个对象。可以随时替换或移除。管道格子不能放置任何对象。

### 转向器如何影响弹球

转向器有 4 种朝向（右上/右下/左上/左下）。只有连通方向进入时才生效，否则弹球反弹。连通时弹球方向被偏转，同时获得等级速度加成（1~5 级 × 1.5x/2.5x/3.5x/4.5x/6x）。旋转器与转向器规则相同，但弹球离开后旋转器会自转一个方位。

### 武器尾巴如何触发充能

武器本体是正方形，占用一格，弹球不能进入。武器的尾巴在本体各朝向反方向一格，最多 4 个方向（上下左右）。弹球经过任意尾巴格时充能 +1（尾巴可以伸到管道格子上，管道中经过同样触发）。充能达到各自上限（手枪 2/炸弹 3/激光 4/闪电 5）后自动开火，充能归零，控制台输出日志（如"手枪 发射"）。开火不影响弹球移动。本版本不做视觉特效。

### 棋盘状态如何变化

弹球移动过程中：经过减速器步长变慢、进入混沌门方向随机化、进入黑洞传送回入口。冰块被弹球撞击扣耐久（初始 10），归零后消失。石头被弹球撞击反弹但不损耗，永不消失，无法被武器破坏。武器开火清除棋盘上的残骸并扣冰块耐久。转向器状态固定不变，旋转器会在弹球离开后自转。

### 后续新增棋盘对象时必须遵守的规则

1. 新对象只能放在内棋盘（行 1~5, 列 1~5），管道格不可放置
2. 必须定义该对象是"可通过"还是"阻挡"
3. 如果可通过，必须定义弹球进入时的效果（改向/变速/传送/充能等）
4. 如果阻挡，必须定义弹球碰到时的效果（反弹/扣耐久/触发等）
5. 每格最多一个对象，新对象替换旧对象
6. 弹球入口坐标可配置（当前暂定行 3, 列 0），同一局内不可改变
7. 管道路径（顺时针 24 格）不可改变
8. 弹球 5 个推进阶段（格效果→移动→武器→离开→历史）的顺序不可改变
9. 武器合成规则：两个同类型同级武器合成 → 等级 +1；尾巴同向则保留 1 条，尾巴异向则保留 2 条；最多 4 条尾巴（上下左右各一）
